<span data-ttu-id="0dd27-111">Pokud je strategie prognózy nastavena na **Kopírování mezi historickými poptávkami**, konec historického horizontu je ignorován.</span><span class="sxs-lookup"><span data-stu-id="0dd27-111">When the forecast strategy is set to **Copy over historical demand**, the end of the historical horizon is ignored.</span></span> <span data-ttu-id="0dd27-112">Systém zkopíruje počet intervalů uvedených v poli **Horizont prognózy** do prognózy poptávky, začínaje datem v poli **Od data** v nabídce **Historický horizont**.</span><span class="sxs-lookup"><span data-stu-id="0dd27-112">The system copies the number of buckets specified in the **Forecast horizon** field to the forecast demand, starting from the date set in the **From date** field under **Historical horizon**.</span></span> <span data-ttu-id="0dd27-113">Díky zkopírování historické poptávky z určitého data dopředu může plánovač výroby vytvořit plán pro další čtvrtletí, a to dvěma způsoby:</span><span class="sxs-lookup"><span data-stu-id="0dd27-113">By copying historical demand from a certain date forward, production planners can make the plan for the next quarter in two ways:</span></span>

-   <span data-ttu-id="0dd27-115">Zkopírováním poptávky z předchozího čtvrtletí.</span><span class="sxs-lookup"><span data-stu-id="0dd27-115">By copying the demand from the previous quarter.</span></span>

<span data-ttu-id="0dd27-116">Aby se zabránilo nejasnostem v plánování výroby, může být určitý počet intervalů prognóz zamrazen.</span><span class="sxs-lookup"><span data-stu-id="0dd27-116">To prevent confusion in the production plans, a certain number of forecast buckets can be frozen.</span></span> <span data-ttu-id="0dd27-117">Tento počet je nastaven v poli **Ochranná doba zablokování**.</span><span class="sxs-lookup"><span data-stu-id="0dd27-117">This number is set in the **Freeze time fence** field.</span></span> <span data-ttu-id="0dd27-118">Na stránce **Upravená prognóza poptávky** jsou buňky pro zmrazené období zakázány a poskytují tak vizuální označení, že by tyto hodnoty neměly být změněny.</span><span class="sxs-lookup"><span data-stu-id="0dd27-118">On the **Adjusted demand forecast** page, the cells for the frozen buckets are disabled, to give a visual indication that those values should not be changed.</span></span>

> [!NOTE]
> <span data-ttu-id="0dd27-125">Náhradu chybějící hodnoty lze použít pouze pro mezery v datech mezi počátečním a koncovým datem historických dat.</span><span class="sxs-lookup"><span data-stu-id="0dd27-125">Missing value substitution only works for the gaps in data in between the start and end dates for historical data.</span></span> <span data-ttu-id="0dd27-126">Data nebudou vyplněna před nebo za posledním fyzickým datovým bodem, jedná se pouze o extrapolaci mezi skutečnými existujícími datovými body.</span><span class="sxs-lookup"><span data-stu-id="0dd27-126">It will not fill in data before or after the last physical data point, it only acts as extrapolation between actual existing data points.</span></span>

<span data-ttu-id="0dd27-141">Vnitropodniková plánovací skupina, alokační klíče položky a dalších filtry lze použít v době generování předpovědi.</span><span class="sxs-lookup"><span data-stu-id="0dd27-141">The intercompany planning group, item allocation keys, and other filters can be applied at forecast generation time.</span></span> <span data-ttu-id="0dd27-142">Tyto pomáhají zvýšit výkon nebo rozdělit data do spravovatelných segmentů.</span><span class="sxs-lookup"><span data-stu-id="0dd27-142">These can be used to improve performance or to split the data into manageable chunks.</span></span> <span data-ttu-id="0dd27-143">Mějte však na paměti, že prognóza poptávky se nevygeneruje pro členy alokačního klíče položky, který není spojen se skupinou mezipodnikového plánování, ani když je alokační klíč položky vybrán v dotazu.</span><span class="sxs-lookup"><span data-stu-id="0dd27-143">However, note that a demand forecast is not generated for the members of any item allocation key that is not associated with an intercompany planning group, even if the item allocation key is selected in the query.</span></span>
